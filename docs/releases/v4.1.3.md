# v4.1.3 Release Notes

Status: Candidate

Target tag: v4.1.3

## Summary

`v4.1.3` adds cycle-context-aware last-weight lookup for workout cards. New
saved weights include the scheduled program/cycle/day context, allowing the app
to prefer a previous load from the same part of the 12-week cycle before
falling back to the latest same-exercise record.

## User-Visible Changes

- Show `Last from Cycle Week X/12 • Day Y` when an exact cycle-context record
  exists.
- Show `Last logged for exercise` when only a legacy or wrong-context record is
  available.
- Keep `No previous` for exercises without saved history.
- Save missed-workout weights with the missed workout's scheduled context.
- Preserve cycle-context fields in History backup export and import.
- Keep old History backups and old saved records readable.

## Implementation Notes

- IndexedDB upgrades from version `1` to version `2`.
- Database name remains `hiit-app-db` and the object store remains `Weights`.
- Existing indexes stay in place, with a new `exerciseCycleContext` compound
  index on `["id", "cycleLength", "cycleWeek", "workoutDay"]`.
- Existing records are not rewritten during upgrade.
- Exact cycle context is available only for records saved after this feature or
  imported from backups that already include valid context fields.
- Quick fill remains out of scope.

## Data And PWA Compatibility

The saved-weight record shape remains backward-compatible:

- Required fields: `{ id, weight, date }`
- Optional context fields: `programWeek`, `cycle`, `cycleWeek`, `cycleLength`,
  and `workoutDay`
- Backup format version: `1`
- Legacy PWA bridge: retained at `/service-worker.js`
- Generated Workbox worker: retained at `/sw.js`

Rollback must account for browsers that have already opened IndexedDB version
`2`; reverting only `DB_VERSION` is unsafe for those clients.
