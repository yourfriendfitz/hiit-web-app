# v4.1.0 Release Notes

Status: Published

Published tag: v4.1.0

## Summary

`v4.1.0` makes the expanded 96-week workout plan easier to understand by
presenting it as eight repeated 12-week cycles. Workout headers stay compact
while the Directory carries the fuller cycle context. This release also refreshes
the app icon set with a simpler dark-and-mint dumbbell mark and a reproducible
icon-generation script.

## User-Visible Changes

- Show scheduled workouts with cycle-relative labels such as
  `Cycle Week 12/12 • Day 5`.
- Show missed-workout metadata with the date plus cycle-relative position, such
  as `Fri, Aug 1 • Cycle Week 1/12 • Day 5`.
- Group the Directory into cycle sections so the full 96-week program reads as
  eight 12-week cycles instead of one long flat list.
- Collapse all Directory cycles by default except the cycle that contains the
  current week.
- Collapse all Directory weeks by default except the current week inside the
  active cycle.
- Replace the old pixel-style favicon and install icons with a cleaner
  brand-aligned dumbbell mark.

## Implementation Notes

- The smallest exact repeated block in `public/data.json` is 12 weeks, so the
  96-week plan is displayed as eight cycles of 12 weeks.
- Cycle calculations live in `src/program-schedule.ts` through
  `PROGRAM_CYCLE_LENGTH_WEEKS`, `getProgramCyclePosition`, and
  `formatWorkoutScheduleLabel`.
- Directory cycle and week collapse behavior lives in
  `src/components/directory.tsx` and uses native `details`/`summary` controls.
- Home missed-workout metadata lives in `src/components/home-workout.tsx`.
- Route subtitles remain coordinated by `src/App.tsx`.
- Icon assets are generated by `npm run generate:icons`, which runs
  `scripts/generate-icons.mjs`.
- `index.html` links the ICO, SVG, 32px PNG, 16px PNG, and Apple touch icon
  explicitly.
- `scripts/assert-pwa-build.mjs` now requires `icon.svg` in the generated PWA
  build.

## Data And PWA Compatibility

No IndexedDB schema or saved-weight migration is included:

- Database: `hiit-app-db`
- Database version: `1`
- Store: `Weights`
- Record shape: `{ id, weight, date }`
- Workout routes: still use global zero-based week and day parameters
- Legacy PWA bridge: retained at `/service-worker.js`
- Generated Workbox worker: retained at `/sw.js`

The plan data is longer, but the storage contract for saved weights is
unchanged because saved records remain keyed to exercise IDs and dates, not to a
program-cycle schema.
